# ADR 0002: 采用指纹库 TTL 与增量分页交互机制

## 状态

Accepted

## 背景

为实现高吞吐量岗位检索且确保不重复，需设计一套轻量级去重机制，并需平衡 API 消耗与人工交互效率。

## 决策

1. **指纹库机制**: 引入 `seen_jobs.json` 索引，存储 `(公司名 + 职位名 + 地点)` 作为唯一指纹，并附带 `last_seen_at` 时间戳。
2. **TTL 衰减**: 指纹有效期设为 30 天。超过 30 天的指纹在搜索时将触发重新检索与内容比对。
3. **分页交互**: 交互强制执行“每 20 个岗位暂停”的阻塞式确认模式，防止 API 无限运行浪费资源。
4. **增量写入**: Markdown 文件采用追加模式，确保分页获取的岗位平滑写入。

## 后果

- **优势**: 有效防止死循环重复检索，节约 API 调用成本，用户主动权增强。
- **代价**: 需维护一个本地索引文件，增加了系统的小额 I/O 开销。
