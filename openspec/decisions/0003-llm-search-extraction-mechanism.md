# ADR 0003: 采用 LLM 驱动的联网检索与提取机制

## 状态

Accepted

## 背景

为降低招聘平台反爬维护成本并提升数据提取效率，放弃传统的硬编码爬虫（Crawler），转而采用搜索引擎加持的 LLM 提取模式。

## 决策

1. **检索实现**: 使用 DeepSeek 的联网功能调用通用搜索引擎（如 Bing/Google）获取岗位信息。
2. **提取实现**: 利用 DeepSeek 的语义理解能力，直接对原始网页摘要进行结构化解析，将其转换为指定的 JSON 格式。
3. **数据校验**: 在代码层引入 Pydantic 模型，对 LLM 输出的 JSON 进行严格的类型与完整性检查。
4. **合规性**: 依托搜索引擎索引，规避直接检索招聘平台的复杂技术与合规风险。

## 后果

- **优势**: 维护成本极低（无需跟进网页布局变更），开发周期短，数据提取语义化程度高。
- **代价**: 依赖 LLM 输出的稳定性，需在 Prompt 中通过指令强制约束 JSON 输出格式。
