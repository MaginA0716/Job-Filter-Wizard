# ADR 0004: 引入 Serper.dev 联网搜索以确保数据真实性

## 状态

Accepted

## 背景

在核心功能验证阶段，发现单纯依赖 LLM (DeepSeek) 的联网功能在检索特定岗位时存在以下局限：

1. **数据时效性不可控**：LLM 内部的搜索结果可能存在延迟。
2. **结构化提取率波动**：当 LLM 直接访问网页时，由于反爬限制，经常无法获取完整的职位列表。
3. **幻觉风险**：在无搜索结果时，LLM 可能会尝试“生成”看似真实的职位。

## 决策

1. **引入 Serper.dev API**：将原本由 LLM 驱动的黑盒搜索逻辑，重构为通过 [Serper.dev](https://serper.dev/) (Google Search API) 获取实时数据。
2. **双层架构逻辑**：
   - **第一层 (Serper)**：获取原始、真实的网页标题、摘要及链接。
   - **第二层 (DeepSeek)**：将 Serper 返回的真实搜索结果作为上下文，交给 LLM 进行精准的 JSON 结构化提取。
3. **环境变量管理**：引入 `SERPER_API_KEY`，并保持 `DEEPSEEK_API_KEY` 仅负责推理逻辑。

## 后果

- **优势**:
  - **真实性大幅提升**：每一个职位均有据可查，有效杜绝了 LLM 幻觉。
  - **搜索精度增强**：利用 Google 强大的搜索语法，过滤更加精准。
- **代价**:
  - 增加了对第三方付费 API (Serper) 的依赖。
  - 每次检索流程增加了 1 次额外的 HTTP 请求。
