# 核心业务逻辑规范 (Core Logic Specification)

## 1. 搜索与去重策略 (Search & Deduplication)

- **指纹定义**: `公司名` + `职位名` + `工作地点`。
- **索引机制**: 使用 `seen_jobs.json` 记录已检索指纹及其 `last_seen_at` 时间戳。
- **检索判定**:
  - 若指纹不存在：发起搜索。
  - 若指纹存在且 `last_seen_at` < 30 天：跳过。
  - 若指纹存在且 `last_seen_at` >= 30 天：重新检索并执行内容比对。
- **内容比对**: 若重检索结果与 30 天前描述完全一致，则仅更新时间戳，不写入新文件；否则视为更新岗位。
- **时效偏好**: 优先检索 7 天内发布的新岗位。

## 2. 数据检索与处理 (Search & Extraction)

- **模型调用**: 使用 DeepSeek API 配合联网功能。
- **结构化输出**:
  - 单次上限：20 个岗位/批次。
  - 强制字段：岗位类型、薪资范围、所属行业、企业类型、人员规模、所需技能/任职要求。
  - 容错处理：缺失字段记为 "N/A"。

## 3. 存储与交互 (Storage & Interaction)

- **文件命名**: `[YYYYMMDD]_[搜索条件].md`。
- **文件格式**: Markdown 表格。
- **分页交互**:
  - 每完成 20 个岗位的检索与写入，系统暂停。
  - 交互询问：“是否继续搜索更多岗位？”
  - 确认后继续追加写入，直至用户停止。
