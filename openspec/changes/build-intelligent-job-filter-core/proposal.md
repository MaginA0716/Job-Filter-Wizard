# 提案 (Proposal): 智能化招聘信息抓取与筛选工具

## 1. 核心价值 (Core Value)

构建一个基于 DeepSeek API 的智能招聘信息筛选器。通过严格的地理位置、技能、行业双向过滤（包含/排除）以及发布时间（3 个月内）的筛选，极大提升求职者获取高匹配度职位的效率。

## 2. 核心功能点 (Key Features)

- **精准过滤**: 严格执行“城市”、“岗位/技能”的必填项过滤，以及“薪资”、“行业”的可选过滤。
- **输入参数约束 (Input Constraints)**:
  - **求职城市 (Required)**: 严格执行地理位置过滤（例如输入：成都）。
  - **岗位/技能 (Required)**: 岗位类型（如输入：前端开发）或 核心技能（如：vue），两者必选其一。
  - **薪资要求 (Optional)**: 设定最低薪资门槛。
  - **行业偏好 (Optional)**: 包含（心仪行业）与 排除（负面行业）的双向过滤。
- **发布时间管控**: 自动计算当前日期，强制剔除 > 3 个月的过时信息。
- **指纹去重**: 扫描历史 Markdown 存储，提取（公司名 + 职位名）作为唯一指纹，实现增量抓取。
- **结构化输出**: 生成 `[YYYYMMDD]_[搜索条件].md` 格式的 Markdown 表格，每次返回 20 个岗位。
- **交互流控制**: 完成单轮（20 个）搜索后系统必须暂停并询问是否继续。

## 3. 验收标准 (Acceptance Criteria)

- [ ] 系统能够成功读取历史 `.md` 文件并建立去重索引。
- [ ] 搜索结果严格符合地理位置与时间窗口要求。
- [ ] 输出文件名符合命名规范且格式为 Markdown 表格。
- [ ] 交互流实现“每 20 个暂停一次”的逻辑。
